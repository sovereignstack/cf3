# CLAUDE.md — working context for Tread

This file is the standing brief for any AI assistant working on this codebase. Read it before making changes, and keep changes consistent with it. If a request conflicts with the **Hard constraints** below, flag the conflict instead of silently breaking them.

## What this is

Tread is a privacy-first, offline-first Progressive Web App that helps one person understand, track, and reduce their personal carbon footprint. It is **India-first** (₹, LPG, local trains, two-wheelers, the Indian grid factor). It is built for a vibe-coding hackathon, so it optimises for: offline robustness, airtight privacy, engagement (gamification), and accessibility.

The entire app is a **single `index.html`** (HTML + CSS + vanilla JS) plus a service worker, a manifest, and icons. There is **no framework, no build step, and no runtime dependency.**

## Hard constraints (do not break these)

1. **No dependencies in the shipped app.** No React/Vue, no bundler, no CDN `<script>`/`<link>`, no npm packages loaded at runtime. Plain vanilla JS only. (A dev-only test harness under `dev/` may use jsdom — that never ships.)
2. **No build step.** The app must run by serving the static files as-is.
3. **No network calls for user data — ever.** The app makes zero `fetch`/`XMLHttpRequest` for user data, and there are **no external/third-party URLs** anywhere. No analytics, telemetry, ad SDKs, fonts-from-CDN, or trackers. The *only* `fetch` in the project is in `sw.js`, caching the app's own same-origin files for offline use. This is a load-bearing privacy guarantee — a reviewer can grep the source to verify it. Keep it true.
4. **No server, no backend, no account, no login.** All state lives in `localStorage` under the single key `tread`, via `load()`/`save()`. The app must stay reachable from a plain static URL.
5. **Light theme only.** No dark mode, no dark-mode toggle, no `prefers-color-scheme: dark` styles. There is a high-contrast *light* variant (`html.hc`) for accessibility — that's the only theme variant.
6. **India-first.** Defaults, units, currency (₹), emission factors, and benchmarks are India-calibrated. Don't switch to US units/factors.
7. **Accessibility is a baseline, not optional.** Preserve: semantic landmarks/headings, real `<label>`s, visible focus rings, the `aria-live` region, keyboard operability, the modal focus trap, the text-size / high-contrast / reduce-motion settings, and **every chart's `sr-only` `<table>` fallback**. Never use colour as the only way to convey meaning.
8. **No-guilt tone.** Copy is calm, second-person, and encouraging. Increases are shown in **amber, never red** (red is reserved for destructive actions like delete). Always pair carbon numbers with tangible equivalences, and always be honest that figures are **estimates**.
9. **Stay lean.** Fewer lines = fewer bugs and a smaller surface for a code reviewer. Prefer the simplest thing that works. Don't add abstraction the app doesn't need.
10. **Relative paths only.** All asset references are relative (`icon-64.png`, `sw.js`, …) so the app works when deployed in a subfolder (e.g. GitHub Pages project sites). Don't introduce absolute `/…` paths.

## Run and test

```bash
# Serve locally (service worker needs HTTPS or localhost, not file://):
python3 -m http.server 8000      # open http://localhost:8000
# or: npx serve

# Run the headless smoke test (catches runtime errors a syntax check can't):
cd dev && npm install && npm test
```

The smoke test (`dev/smoke-test.js`) loads `index.html` in jsdom and drives the real flow: consent → quiz → log → demo data → delete/undo → tab navigation → settings → pledge. **Run it after every change.** Also lint quickly by extracting the inline script and running `node --check` on it.

### Definition of done for any change
- `dev/npm test` passes and the inline script passes `node --check`.
- Keyboard-only operation still works; charts still expose their data tables; focus is visible.
- The app still works fully offline. **If you change which files are cached, update the `SHELL` array *and* bump `VERSION` in `sw.js`** (otherwise users get stale cached files).
- No new dependency, no network call for data, no external URL, no dark mode.
- Copy stays no-guilt and honest about estimates.

## Architecture of `index.html`

Everything is in one file, organised in this order inside the single `<script>`:

- **Helpers:** `$`, `$$`, `live()` (announces to the `aria-live` region).
- **Data tables (constants):** `F` (emission factors), `SUB`/`CATLABEL`/`CATCOLOR`/`CATEM` (labels), `BM` (benchmarks), `PRESETS` (quick-log chips), `ACTIONS` (suggestions), `LEVELS`, `BADGES`.
- **Storage:** `blank()` returns the default state object; `load()`/`save()` read/write `localStorage["tread"]`. The whole app state is one object `S`.
- **Date helpers:** `todayISO()` (local date, **not** UTC — keep it local), `monthKey()`, `monthLabel()`, `lastMonths()`, `niceDate()`.
- **Engine (pure-ish):** `kgFor()`, `logsIn()`, `sumKg()`, `byCat()`, `equiv()`, `fmt()`, `monthFigure()` (real logs if any this month, else the quiz baseline ÷ 12, flagged `est:true`), `baselineFrom()` (quiz → annual estimate by category).
- **Gamification:** `recomputeProgress()` — **the single source of truth** for XP, streak, best, and badges. It recomputes from the full `S.logs` + `S.pledges`. **Call it after any change to logs or pledges** (add, delete, demo add/remove, pledge, mark-done). Do not increment XP/streak ad hoc elsewhere.
- **Logs:** `addLog()`, `removeLog()`, demo data (`addDemoData()`/`removeDemoData()`/`pushDemo()`), and log-list rendering (`renderLogList()`, `logRow()`, `deleteLog()`, `bindDelete()`, `openAllLogs()`).
- **Router:** `route()` decides the first screen (welcome if no consent → quiz if no baseline → current tab). `show(view, showChrome)` toggles `.screen.active`, app bar, and bottom nav, then calls the matching `render*()` and moves focus to the screen's `<h1>`. `go(view)` is the tab switch.
- **Charts (return SVG strings):** `donut()`, `hbars()`, `trend()`. **Each must include an `sr-only` `<table>`** with the same numbers and an `aria-label` summary.
- **Screens:** `renderHome()`, `renderInsights()`, `renderAct()`, `renderYou()`. Each sets the innerHTML of its `#*-dynamic` container, then binds handlers (e.g. `bindGo`, `bindPledge`, `bindDelete`).
- **Log sheet & modals:** `openLog()`, `openDetailed()`, `openSheet()`/`closeSheet()` (with focus trap + `Esc`), `confirmSheet()`.
- **Toast:** `toast(msg, undoFn)` — transient status with an optional **undo callback** (used by add-log and delete).
- **Data actions:** `exportData()` (download JSON), `wipeAll()` (erase).
- **Settings:** `applySettings()` sets `document.documentElement.className` to `ts-<size>` plus optional `hc`/`rm`.
- **Wiring + boot:** the privacy notice text, consent handlers, quiz handlers, nav/log wiring, then `preselQuiz(); applySettings(); route();` and the service-worker registration + online/offline indicator.

### State shape (`S`, persisted as `localStorage["tread"]`)
```js
{
  consent: { given, version, ts } | null,
  profile: { createdAt, baselineAnnualKg, byCat:{travel,food,home,shopping}, house,
             xp, streak, best, lastLog },
  logs: [ { id, cat, sub, qty, unit, kg, dateISO, ts, demo? } ],
  pledges: { [actionId]: { status:'active'|'done', ts } },
  badges:  { [badgeId]: dateISO },
  settings:{ textSize:'s'|'m'|'l'|'xl', hc:bool, rm:bool|'system', region:'IN' }
}
```

## Emission factors & methodology

Factors live in the `F` object (transport kg/km, food kg/meal, home kg/kWh and kg/kg, shopping kg/item) and in the maps inside `baselineFrom()`. They are **published averages, i.e. estimates** — keep them honest, keep the in-app *About* text, `README.md`, and `PRD.md` in sync when you change them, and cite sources (CEA grid factor for India ≈ 0.71 kg/kWh; Poore & Nemecek 2018 for food; standard per-km transport factors; a ~1.9× radiative-forcing multiplier folded into the flight factor). Benchmarks in `BM` are India-calibrated (India ≈ 1.9 t/yr, target ≈ 1.5 t/yr, global ≈ 4.7 t/yr), expressed per month.

## Gotchas already learned (don't reintroduce)

- **Event-as-argument bug:** never write `el.onclick = fnThatTakesArgs`. The DOM passes the event as the first argument. Use `el.onclick = () => fn(arg)`. (This exact bug broke the detailed-entry sheet once.)
- **Re-view before editing:** after an edit, the file's line numbers shift; re-read before the next edit.
- **CSS fallbacks:** `color-mix()` and `backdrop-filter` have solid-colour and `-webkit-` fallbacks in the app bar / nav. Keep them for older browsers.
- **Service worker caching:** bump `VERSION` in `sw.js` whenever cached files change, or testers will keep getting old assets.
- **Consent is not a login:** the first-run welcome is a one-time, on-device consent (one checkbox). Don't turn it into an account or a gate that blocks automated evaluation beyond that single tap.
- **`todayISO()` is local time on purpose** — don't switch to `toISOString()` (UTC) or streaks/dates drift across time zones.

## How to do common tasks

- **Add an emission factor / activity type:** add it to `F[cat]`, add a label to `SUB`, optionally add a `PRESETS` chip and/or an `openDetailed()` option, then update the *About* text + `README.md`/`PRD.md`.
- **Add a quick-log preset:** append to `PRESETS`.
- **Add a suggested action:** append to `ACTIONS` (with `cat`, `effort`, `save` kg/yr, `why`). Ranking in `topActions()` boosts the user's biggest category.
- **Add a screen/tab:** add a `<section class="screen">`, a nav button (or entry point), a `render*()` function, and a branch in `show()`.
- **Change the look:** edit the CSS custom properties in `:root` (and the `html.hc` overrides). Keep contrast ≥ 4.5:1 for text.
